# リサーチワークフロー

**トリガー:** `/research <URLまたはトピック>`

**目的:** 
`antigravity.codes` や `cursor.directory` から、プロジェクトに最適な **Rules**、**Workflows**、**Skills** の技術データを高速かつ網羅的に収集・蓄積する。
※ このワークフロー内では「データ蓄積」のみを行い、プロジェクトへの適用（最適化）は行わない。

---

## 1. 環境準備
- **MCP サーバー起動**: `. .agent/skills/tavily\ /scripts/start-tavily-mcp.sh` が実行されているか、または即座に実行して Tavily MCP を有効化する。
- **API キー確認**: `.env.mcp` の `TAVILY_API_KEY` の存在を確認。

## 2. 高速情報収集 (Tavily MCP)
- **マッピング**: `tavily_map` を使用して、指定されたサイト（例: `https://antigravity.codes/rules`）から Rules/Skills の URL 一覧を秒速で取得する。
- **フィルタリング**: プロジェクトの技術スタック（Next.js, TypeScript, Tailwind等）に関連性の高い URL を特定する。
- **一括抽出**: 抽出対象の URL 群に対し `tavily_extract` を実行し、Markdown 形式のテキストコンテンツを取得する。

## 3. 詳細データ収集 (サブエージェント)
- **詳細スクレイピング**: Tavily で取得しきれない複雑なコードブロックや、動的レンダリングが必要なページについては、`agent-browser` (サブエージェント) を起動して詳細に収集。
- **スクリーンショット保存**: 必要に応じて画像データも収集。

## 4. 構造化保存
- **保存用ディレクトリ**: `.agent/research-data/<domain>/<YYYYMMDD>_<topic>.md` 
- **ファイル構成**: 
    - メタデータ (URL, 取得日, カテゴリ)
    - 取得した原文 (Rules/Skills/Workflows)
    - 簡易的な要約（日本語）

## 5. 完了と停止
- **成果物の提示**: 保存したファイルへのパスを提示。
- **重要命令 (STOP)**: データの収集・蓄積が完了したら、作業を**停止**する。
- **返答**: 「データの収集が完了しました。保存された情報は [リンク] にあります。次に **Pro版モデル** に切り替えて、これらの情報を元にプロジェクトへの最適化（スキル/ルールの作成）を依頼してください。」

---

**システム命令:**
このワークフローは「データ収集」に特化しており、`.agent/rules` や `.agent/skills` の新規作成、または `src/` の編集をこのステップで同時に行うことは**絶対禁止**です。
収集フェーズと活用フェーズを明確に分離し、収集完了後は速やかに停止してください。
